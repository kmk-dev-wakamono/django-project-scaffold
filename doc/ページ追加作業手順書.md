# ページ追加作業手順書

目的: Django アプリ（例: `hello`）に新しい静的または簡単な動的ページを追加し、ローカルで表示確認する手順を記載します。

## 前提

- リポジトリのルートで作業する（例: `C:\Users\zhu2q\repositories\python-project-skeleton`）
- 仮想環境が有効で、`requirements.txt` がインストール済みであること（`uv` を使う場合は該当する `uv` の操作を先に済ませてください）
- Django プロジェクト名は `mysite`、対象アプリは `hello` とする（必要に応じて置換してください）

## 作業手順（概要）

1. ビュー関数を追加する
2. URL パターンを登録する
3. テンプレートを作成する
4. （必要なら）モデル・マイグレーションを作成する
5. 開発サーバで動作確認する
6. （任意）テストを追加して実行する
7. 変更をコミットする

## 詳細手順

### 1. ビュー関数を追加する

ファイル: [hello/views.py](hello/views.py)

例: `about` ページを追加する場合、`hello/views.py` に以下を追加します。

```python
from django.shortcuts import render

def about(request):
    return render(request, 'hello/about.html', {'title': 'About', 'content': 'This is the about page.'})
```

ポイント:

- ビューでデータベースを使わないならシンプルに `render` でテンプレートを返します。

### 2. URL パターンを登録する

ファイル: [hello/urls.py](hello/urls.py)

`urlpatterns` に新しいパスを追加します。

```python
from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='index'),
    path('hello/<str:user>/', views.greet, name='greet'),
    path('about/', views.about, name='about'),  # 追加
]
```

ポイント:

- プロジェクトの `mysite/urls.py` がアプリの URLconf を include していることを確認してください（通常は既に設定済みです）。

### 3. テンプレートを作成する

ディレクトリ: `templates/hello/`

ファイル: `templates/hello/about.html`

```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8">
    <title>{{ title }}</title>
  </head>
  <body>
    <h1>{{ title }}</h1>
    <p>{{ content }}</p>
    <p><a href="/">Back</a></p>
  </body>
</html>
```

ポイント:

- `TEMPLATES` 設定で `BASE_DIR / 'templates'` が有効になっているか確認してください（`mysite/settings.py`）。

### 4. （必要なら）モデルとマイグレーション**

ページがデータベースのデータを使う場合は `hello/models.py` にモデルを追加し、マイグレーションを作成します。

```powershell
python manage.py makemigrations
python manage.py migrate
```

### 5. 開発サーバで動作確認

```powershell
# 仮想環境をアクティブにしたあと
python manage.py runserver
```

ブラウザで以下を確認:

- `http://127.0.0.1:8000/about/` が 200 を返し、意図した内容が表示される

### 6. テスト追加（任意だが推奨）

ファイル: `hello/tests.py`

簡単な例:

```python
from django.test import TestCase

class AboutPageTests(TestCase):
    def test_about_page(self):
        resp = self.client.get('/about/')
        self.assertEqual(resp.status_code, 200)
        self.assertContains(resp, 'About')
```

実行:

```powershell
python manage.py test hello
```

### 7. 変更をコミットする

mainブランチには push できないようになっているので、dev ブランチに切り替え、変更を add, commit してください。

```powershell
git add .
git commit -m "Add about page to hello app"
```

チェックリスト（作業前に確認）

- 仮想環境が有効か
- ブランチは main になっていないか
- `requirements.txt` がインストール済みか
- `mysite/settings.py` の `TEMPLATES` と `INSTALLED_APPS` に問題がないか

トラブルシュートのヒント

- 404 が出る: `mysite/urls.py` が `hello.urls` を include しているか確認
- テンプレートが見つからない: `TEMPLATES['DIRS']` に `BASE_DIR / 'templates'` が入っているか確認
- サーバが立ち上がらない: 依存パッケージがインストールされているか (`pip install -r requirements.txt`)
